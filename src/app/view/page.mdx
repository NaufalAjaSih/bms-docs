## Kapan Menggunakan Modal vs Page View

### Gunakan Modal View ketika:

-   Form input sederhana (1-5 field)
-   Operasi cepat dan tidak memerlukan navigasi
-   User experience yang smooth tanpa reload halaman
-   Data yang tidak terlalu kompleks

### Gunakan Page View ketika:

-   Form input kompleks (banyak field)
-   Memerlukan validasi yang rumit
-   Ada file upload yang besar
-   Memerlukan preview atau preview data
-   Operasi yang memerlukan konfirmasi detail

## Struktur View

### Struktur Folder

```
resources/views/admin/
├── layout.blade.php              # Layout utama admin
├── modal/
│   └── index.blade.php           # CRUD Modal view
└── page/
    ├── index.blade.php           # List view untuk CRUD Page
    ├── create.blade.php          # Create form untuk CRUD Page
    └── show.blade.php            # Edit form untuk CRUD Page
```

### Naming Convention

| Jenis View    | Format             | Contoh                        |
| ------------- | ------------------ | ----------------------------- |
| **Layout**    | `layout.blade.php` | `admin/layout.blade.php`      |
| **Index**     | `index.blade.php`  | `admin/modal/index.blade.php` |
| **Create**    | `create.blade.php` | `admin/page/create.blade.php` |
| **Show/Edit** | `show.blade.php`   | `admin/page/show.blade.php`   |

            spinner.removeClass('d-none');

                                '<span class="invalid-feedback" role="alert"><strong>' +

                            );

                        btnText.text('Simpan');
                        submitBtn.prop('disabled', false);

                }

                                toastr.error("Gagal menghapus data.", "GAGAL!", {
                                    progressBar: true,
                                    timeOut: 3500,
                                    positionClass: "toast-bottom-right"
                                });

                                btnText.innerHTML = `Ya, Hapus`;
                                confirmBtn.disabled = false;
                            }
                        });
                    });
                }
            });
        });

                    },

                });

        });

## JavaScript Standards

### AJAX Form Submission

```javascript
// Form submission
$("#formName").on("submit", function (e) {
    e.preventDefault();

    var formData = new FormData(this);
    var isEdit = $("#formName").data("edit-id");
    var url = isEdit ? updateUrl + "/" + isEdit : createUrl;
    var method = isEdit ? "PUT" : "POST";

    $.ajax({
        url: url,
        type: method,
        data: formData,
        processData: false,
        contentType: false,
        success: function (response) {
            if (response.status === "success") {
                $("#modalName").modal("hide");
                table.ajax.reload();
                $("#formName")[0].reset();
                $("#formName").removeData("edit-id");
                $(".modal-title").text("Add Data");
                toastr.success("Data saved successfully!");
            }
        },
        error: function (xhr) {
            var errors = xhr.responseJSON.errors;
            $.each(errors, function (key, value) {
                toastr.error(value[0]);
            });
        },
    });
});
```

### Delete Confirmation

```javascript
// Delete confirmation
$(document).on("click", ".delete-button", function (e) {
    e.preventDefault();

    var form = $(this).closest("form");
    var url = form.attr("action");

    Swal.fire({
        title: "Are you sure?",
        text: "You won't be able to revert this!",
        icon: "warning",
        showCancelButton: true,
        confirmButtonColor: "#3085d6",
        cancelButtonColor: "#d33",
        confirmButtonText: "Yes, delete it!",
    }).then((result) => {
        if (result.isConfirmed) {
            $.ajax({
                url: url,
                type: "DELETE",
                data: {
                    _token: $('meta[name="csrf-token"]').attr("content"),
                },
                success: function (response) {
                    if (response.status === "success") {
                        table.ajax.reload();
                        toastr.success("Data deleted successfully!");
                    }
                },
                error: function (xhr) {
                    toastr.error("Error deleting data!");
                },
            });
        }
    });
});
```

## Best Practices

### 1. **Layout Consistency**

-   Gunakan layout yang konsisten di semua halaman
-   Implementasikan breadcrumb untuk navigasi
-   Gunakan card container untuk konten

### 2. **Form Validation**

-   Selalu tampilkan error validation
-   Gunakan class `is-invalid` untuk styling error
-   Implementasikan client-side validation

### 3. **User Experience**

-   Gunakan loading indicators
-   Implementasikan confirmation dialogs
-   Berikan feedback yang jelas kepada user

### 4. **Security**

-   Selalu gunakan CSRF token
-   Escape output HTML dengan `e()` helper
-   Validasi file upload

### 5. **Performance**

-   Gunakan lazy loading untuk gambar
-   Minify CSS dan JavaScript
-   Optimasi query database

### 6. **Responsive Design**

-   Gunakan Bootstrap grid system
-   Test di berbagai ukuran layar
-   Implementasikan mobile-first approach
