# Inisialisasi Proyek

Bagian ini menjelaskan langkah-langkah untuk memulai proyek baru berdasarkan **template utama**.
Setiap proyek baru **wajib** dimulai dari repository template agar struktur folder, konfigurasi, dan standar coding tetap konsisten di seluruh proyek tim.

## Langkah-Langkah Inisialisasi

### 1. Clone Repository Template

Clone repository template dari link GitHub ini.

```bash
git clone https://github.com/NaufalAjaSih/boilerplate
```

Masuk ke folder proyek:

```bash
cd boilerplate
```

### 2. Hapus Remote Repository Template

Agar proyek tidak terhubung dengan repository sumber (template), lepaskan remote origin bawaan:

```bash
git remote remove origin
```

Ini penting agar push tidak salah tujuan ke repository template utama.

### 3. Buat Repository Baru di GitHub

Buat repository baru di GitHub (akun pribadi atau organisasi).
Pastikan repository baru **kosong**, tanpa:

- README
- .gitignore
- License

Agar tidak menyebabkan konflik saat push pertama.

Contoh nama repo baru:

```
myapp-laravel
```

### 4. Tambahkan Remote Repository Baru

Tambahkan remote baru yang diarahkan ke repository yang baru dibuat:

```bash
git remote add origin https://github.com/username/myapp-laravel.git
```

Cek remote untuk memastikan:

```bash
git remote -v
```

### 5. Install Dependensi Composer

Setelah repository siap, install semua dependensi Laravel:

```bash
composer install
```

Pastikan Composer sudah terpasang di perangkatmu.

### 6. Salin File Environment

Laravel menyediakan file contoh `.env.example`.
Salin menjadi `.env`:

```bash
cp .env.example .env
```

Jika di Windows:

```powershell
copy .env.example .env
```

### 7. Generate Application Key

Laravel membutuhkan `APP_KEY` untuk enkripsi aplikasi.

Jalankan:

```bash
php artisan key:generate
```

### 8. Migrasi Database

Sebelum melakukan migrasi, pastikan kamu sudah mengatur konfigurasi database di dalam `.env`.

Lalu jalankan migrasi:

```bash
php artisan migrate
```

Jika database belum dibuat, buat database terlebih dahulu melalui phpMyAdmin atau CLI sesuai kebutuhan.

### 9. Jalankan Server Development

Untuk menjalankan server Laravel:

```bash
php artisan serve
```

Server akan berjalan di:

```
http://127.0.0.1:8000
```

## 10. Push Proyek ke Repository Baru

Jika semuanya sudah siap, lakukan push pertama:

```bash
git add .
git commit -m "Initial commit"
git push -u origin main
```

Atau jika menggunakan branch `master`:

```bash
git push -u origin master
```

## Catatan Tambahan

- **Jangan** melakukan commit atau push ke repository template utama.
- Gunakan branch **main** sebagai standar.
- Jika ingin memperbarui template, gunakan **pull request** di repository template untuk menjaga konsistensi dan kontrol perubahan.
- Pastikan setiap anggota tim mengikuti standar workflow Git yang sama.
