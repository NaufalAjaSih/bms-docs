## Standar Penamaan

### Controller

| Jenis                 | Format                    | Contoh                                          |
| --------------------- | ------------------------- | ----------------------------------------------- |
| **CRUD Modal**        | `{Entity}ModalController` | `BiodataModalController`, `UserModalController` |
| **CRUD Page**         | `{Entity}PageController`  | `BiodataPageController`, `UserPageController`   |
| **Entity Controller** | `{Entity}Controller`      | `BiodataController`, `UserController`           |

**Contoh Implementasi:**

```php
// CRUD Modal Controller
class BiodataModalController extends Controller
{
    // Methods untuk CRUD dengan modal
}

// CRUD Page Controller
class BiodataPageController extends Controller
{
    // Methods untuk CRUD dengan halaman terpisah
}

// Entity Controller (untuk API atau custom logic)
class BiodataController extends Controller
{
    // Methods untuk entity Biodata
}
```

### Model

| Aturan         | Format               | Contoh                         |
| -------------- | -------------------- | ------------------------------ |
| **Nama Model** | PascalCase, Singular | `Biodata`, `User`, `Product`   |
| **Table Name** | snake_case, plural   | `biodata`, `users`, `products` |
| **File Name**  | PascalCase           | `Biodata.php`, `User.php`      |

**Contoh Implementasi:**

```php
class Biodata extends Model
{
    protected $table = 'biodata';  // snake_case, plural
    protected $fillable = [
        'nama',
        'jenis_kelamin',
        'tgl_lahir',
        'foto',
    ];
}
```

### Migration

| Aturan           | Format                                      | Contoh                                       |
| ---------------- | ------------------------------------------- | -------------------------------------------- |
| **File Name**    | `{timestamp}_create_{table_name}_table.php` | `2024_01_01_000000_create_biodata_table.php` |
| **Table Name**   | snake_case, plural                          | `biodata`, `users`, `products`               |
| **Column Names** | snake_case                                  | `nama`, `jenis_kelamin`, `tgl_lahir`         |

**Contoh Implementasi:**

```php
Schema::create('biodata', function (Blueprint $table) {
    $table->id();
    $table->string('nama');
    $table->integer('jenis_kelamin');
    $table->date('tgl_lahir');
    $table->string('foto')->nullable();
});
```

### Route

| Jenis              | Format                           | Contoh                                      |
| ------------------ | -------------------------------- | ------------------------------------------- |
| **Resource Route** | `{prefix}/{entity}` (kebab-case) | `admin/biodata-modal`, `admin/biodata-page` |
| **Custom Route**   | `{prefix}/{entity}/{action}`     | `admin/biodata/export`                      |
| **Route Name**     | `{entity}.{action}`              | `biodata-modal.index`, `biodata-page.store` |

**Contoh Implementasi:**

```php
// Resource routes untuk Modal dan Page
Route::resource('admin/biodata-modal', BiodataModalController::class);
Route::resource('admin/biodata-page', BiodataPageController::class);

// Custom routes
Route::get('admin/biodata/export', [BiodataController::class, 'export'])
    ->name('biodata.export');
```

### File Upload

| Jenis             | Format                       | Contoh                                          |
| ----------------- | ---------------------------- | ----------------------------------------------- |
| **Upload Folder** | `public/assets/{entity}/`    | `public/assets/biodata/`                        |

```php
$filePath = 'assets/biodata/' . $filename;
```

## Perbedaan CRUD Modal vs CRUD Page

### CRUD Modal

-   **Controller**: `{Entity}ModalController`
-   **View**: `admin/{entity}/modal/index.blade.php`
-   **Route**: `admin/{entity}-modal`
-   **Fitur**: Semua operasi CRUD dalam modal
-   **UX**: Single page application feel

### CRUD Page

-   **Controller**: `{Entity}PageController`
-   **View**: `admin/{entity}/page/` (index, create, show)
-   **Route**: `admin/{entity}-page`
-   **Fitur**: Halaman terpisah untuk setiap operasi
-   **UX**: Traditional multi-page application

## Checklist Struktur

### Folder Structure

-   [ ] Controller di `app/Http/Controllers/`
-   [ ] Model di `app/Models/`
-   [ ] Migration di `database/migrations/`
-   [ ] View di `resources/views/admin/{entity}/`
-   [ ] Upload folder di `public/assets/{entity}/`

### Naming Convention

-   [ ] Controller menggunakan PascalCase tanpa prefix CRUD
-   [ ] Model menggunakan PascalCase
-   [ ] Migration menggunakan snake_case
-   [ ] Route menggunakan kebab-case dengan suffix modal/page
-   [ ] View menggunakan snake_case dengan struktur modal/page

### File Organization

-   [ ] CRUD Modal dan Page terpisah dengan jelas
-   [ ] Layout konsisten
-   [ ] Asset folder terorganisir per entity
-   [ ] Route terstruktur dengan suffix yang jelas

## Best Practices

1. **Konsistensi** - Selalu ikuti standar penamaan yang telah ditetapkan
2. **Deskriptif** - Gunakan nama yang jelas dan mudah dipahami
3. **Singular/Plural** - Model singular, table plural
4. **Hierarchy** - Organisir file berdasarkan fungsi dan tipe
5. **Scalability** - Struktur yang dapat dikembangkan dengan mudah
6. **Separation** - Pisahkan dengan jelas antara Modal dan Page CRUD
